1. ¿Qué hace esta ruta?
Esta ruta está configurada bajo el método HTTP POST en la dirección '/reporte'. A diferencia de las rutas GET, no está diseñada para entregar páginas o textos directamente al navegador, sino para recibir información segura o estructurada que envía un cliente en formato JSON (en este caso, un objeto que contiene un 'mensaje'). Cuando el servidor procesa con éxito estos datos, devuelve una confirmación en formato JSON indicando que el estado es "Reporte recibido" junto con el mensaje enviado.

2. ¿Para qué podría servir en una aplicación comunitaria?
En el contexto de una aplicación comunitaria, esta ruta es fundamental para la participación ciudadana y la gestión del vecindario. Podría servir para que los habitantes envíen reportes en tiempo real sobre daños en la infraestructura pública (como luminarias rotas o baches en las calles), registrar solicitudes de servicios públicos, o enviar alertas de seguridad vecinal. Al ser una ruta POST, permite empaquetar de forma organizada datos detallados, descripciones del problema o ubicaciones, enviándolos directamente a una base de datos para que los coordinadores comunitarios los procesen de inmediato.
